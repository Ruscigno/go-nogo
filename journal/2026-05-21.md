### Decisions made

- **Go/No-Go gets a Go backend.** It polls the NWS Aviation Weather
  Center API on a schedule, parses METAR/TAF, evaluates verdicts, emails
  on verdict change, and generates WINGS PDFs — all of which the
  founder's backend rule assigns to a Go service on Cloud Run. The web
  tier stays SvelteKit. Formal call recorded in ADR-0001 (proposed).
- **The weather source is the free public NWS Aviation Weather Center
  API** (ADR-0002) — no commercial weather licence; treated as both a
  trust boundary and an availability dependency.
- **The verdict-evaluation engine is a pure function** (ADR-0003) — the
  product's correctness moat; `(parsedWeather, minimums) -> verdict`,
  clock-injected, table-tested. Mirrors currency-hub's pure-function
  currency engine.
- **The verdict never defaults to green** on missing/stale/unparseable
  weather — a safety invariant baked into the engineering + security
  rules and the weather-and-verdict-auditor.
- **Alert dedupe is a DB UNIQUE constraint** (ADR-0004), per-recipient,
  keyed by the verdict-transition identity. **The alert channel is
  email** (ADR-0004) — the portfolio cut Web Push from V1 (iOS PWA
  limitation); no SMS.
- **The weather poll runs on a fixed cadence with a shared-observation
  cache** (ADR-0005) — polls only active-saved-trip airports, caches by
  `(station, issued_at)`, respects an NWS-friendly request budget.
- **Billing is Stripe Checkout + Portal, $6/mo + $39/yr** (ADR-0006).
- Three product-specific auditor subagents created alongside
  `spec-guardian`: `weather-and-verdict-auditor`,
  `alert-pipeline-auditor`, `rls-and-tenancy-auditor`.
- The aviation-domain disclaimer is calibrated **firm** — closer to
  currency-hub's middle-bar, because a go/no-go verdict speaks directly
  to a safety-of-flight decision.

### Deferred
